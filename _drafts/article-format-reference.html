建站笔记

用 GitHub Pages 和 Hexo 搭建自己的博客

发布时间：2026-06-10 · 约 5 分钟阅读

这篇示例文章按照参考资料的思路，保留了最关键的建站路径：先准备本地环境，再用 Hexo 生成博客，最后把静态文件部署到 GitHub Pages。

1. 准备工具

安装 Node.js、Git 和一个代码编辑器。确认命令行可以使用 node、npm 和 git 后，再安装 Hexo。

npm install -g hexo-cli
hexo init my-blog
cd my-blog
npm install

2. 本地预览

在博客目录中启动预览服务，打开浏览器查看默认主题。如果能看到页面，就说明基础环境已经走通。

hexo clean
hexo generate
hexo server

3. 发布第一篇文章

Hexo 的文章通常保存在 source/_posts。你可以用命令生成草稿，再用 Markdown 写正文。

hexo new "我的第一篇博客"

4. 部署到 GitHub Pages

创建名为 Phillin-lrz.github.io 的仓库，安装部署插件，并在 Hexo 配置文件中填写仓库地址。

npm install hexo-deployer-git --save

然后在 _config.yml 中配置：

deploy:
  type: git
  repo: https://github.com/Phillin-lrz/Phillin-lrz.github.io.git
  branch: main

最后执行部署命令：

hexo clean
hexo deploy

5. 下一步

完成上线后，可以继续配置主题、头像、社交链接、评论系统、统计工具和自定义域名。先让写作流程稳定下来，再慢慢打磨视觉效果，会更容易坚持。
